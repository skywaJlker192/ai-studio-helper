## Описание проекта

AI Study Helper - это веб-приложение для изучения различных предметов через систему вопросов и ответов. Проект разработан на React с использованием современных технологий веб-разработки. Приложение позволяет пользователям генерировать вопросы по различным темам, проходить квизы, тренироваться в режиме скорости и отслеживать свой прогресс обучения.

## Технологический стек

- **Frontend Framework**: React 18.2.0
- **Роутинг**: React Router DOM 6.20.0
- **Анимации**: Framer Motion 10.16.16
- **Стилизация**: Tailwind CSS 3.3.6
- **Иконки**: Lucide React 0.294.0
- **Аутентификация**: React OAuth Google 0.12.2
- **Сборщик**: Vite 5.0.8

## Соблюдение принципов ООП

В проекте используются принципы объектно-ориентированного программирования, несмотря на то, что React преимущественно функциональный фреймворк. Основная демонстрация ООП реализована в файле `src/utils/QuestionCache.js`:

### Инкапсуляция
Класс `QuestionCache` использует приватные поля (синтаксис `#`) для скрытия внутренней реализации:
```javascript
class QuestionCache {
  #cache = new Map()
  #maxSize
  #ttl
  #hits = 0
  #misses = 0
}
```
Приватные методы `#evictOldest()` скрывают логику удаления устаревших записей от внешнего кода.

### Наследование
Класс `PriorityQuestionCache` наследуется от `QuestionCache`, расширяя его функциональность:
```javascript
export class PriorityQuestionCache extends QuestionCache {
  #priorities = new Map()
  
  constructor(maxSize, ttl) {
    super(maxSize, ttl)  // Вызов конструктора родительского класса
  }
}
```
Это демонстрирует принцип наследования - дочерний класс получает все свойства и методы родительского класса и может добавлять новые.

### Полиморфизм
Метод `_evictOldest()` переопределен в `PriorityQuestionCache` для реализации собственной логики удаления с учетом приоритетов, что демонстрирует полиморфизм - один интерфейс, разные реализации.

### Абстракция
Классы скрывают сложность работы с кэшем за простым API (методы `set()`, `get()`, `has()`, `clear()`), что является примером абстракции.

## Алгоритмы и их сложность

В проекте реализовано множество алгоритмов различной сложности в файле `src/utils/algorithms.js`:

### 1. Быстрая сортировка (Quick Sort)
**Сложность**: O(n log n) в среднем, O(n²) в худшем случае
**Использование**: Сортировка массивов вопросов, статистических данных

### 2. Бинарный поиск (Binary Search)
**Сложность**: O(log n)
**Использование**: Поиск вопросов в отсортированных массивах, поиск тем по ID

### 3. Поиск подстроки (Boyer-Moore упрощенная версия)
**Сложность**: O(n*m) в худшем случае, O(n/m) в лучшем
**Использование**: Поиск ключевых слов в ответах пользователей

### 4. Полиномиальный хеш строки
**Сложность**: O(n)
**Использование**: Хеширование вопросов для быстрого поиска, создание уникальных идентификаторов

### 5. Наибольшая общая подпоследовательность (LCS)
**Сложность**: O(n*m)
**Использование**: Сравнение ответов пользователей с правильными ответами, определение схожести текстов

### 6. Алгоритм Дейкстры
**Сложность**: O(V²) или O(E log V) с приоритетной очередью
**Использование**: Поиск оптимальных путей в графе тем и категорий

### 7. Обход графа в глубину (DFS)
**Сложность**: O(V + E)
**Использование**: Обход структуры тем и категорий, поиск связанных вопросов

### 8. Обход графа в ширину (BFS)
**Сложность**: O(V + E)
**Использование**: Поиск вопросов по уровням сложности, навигация по иерархии тем

### 9. Проверка на палиндром
**Сложность**: O(n)
**Использование**: Валидация ответов, проверка корректности ввода

### 10. Частотный анализ
**Сложность**: O(n)
**Использование**: Анализ статистики ответов, определение наиболее частых ошибок

### Дополнительные алгоритмы в проекте:

**Алгоритм Фишера-Йетса** (в `rubqLoader.js`) - O(n) для перемешивания вопросов
**Алгоритм нормализации текста** (в `QuestionGenerator.jsx`) - O(n) для приведения ответов к единому формату
**Алгоритм проверки ответов по ключевым словам** (в `QuestionGenerator.jsx`) - O(n*m) для сравнения ответов
**Алгоритм кэширования LRU** (в `QuestionCache.js`) - O(1) для доступа, O(n) для удаления старых записей

## Наличие удобного, адекватного и хорошего GUI интерфейса

Интерфейс приложения разработан с использованием современных принципов UI/UX дизайна:

### Визуальный дизайн
- **Glassmorphism эффекты**: Полупрозрачные карточки с размытием фона создают современный внешний вид
- **Градиенты**: Использование цветовых градиентов для акцентов и кнопок
- **Анимации**: Плавные переходы и анимации с использованием Framer Motion для улучшения пользовательского опыта
- **Адаптивность**: Полностью адаптивный дизайн для различных размеров экранов (мобильные, планшеты, десктопы)

### Навигация
- **Интуитивная навигация**: Четкая структура меню с иконками и подписями
- **Хлебные крошки**: Понятная навигация по разделам приложения
- **Быстрый доступ**: Статус-бар с монетами и уровнем всегда виден пользователю

### Интерактивность
- **Hover эффекты**: Визуальная обратная связь при наведении на элементы
- **Анимации состояний**: Плавные переходы между состояниями компонентов
- **Модальные окна**: Для важных действий (аутентификация, подтверждения)
- **Прогресс-бары**: Визуальное отображение прогресса обучения и уровней

### Функциональность интерфейса
- **Генератор вопросов**: Удобный интерфейс с выбором темы, категории и количества вопросов
- **Режим квиза**: Интуитивный интерфейс с таймером и визуальной обратной связью
- **Режим скорости**: Оптимизированный для быстрого ввода ответов
- **Статистика**: Информативные графики и диаграммы для отслеживания прогресса
- **Система подсказок**: Индивидуальные подсказки для каждого вопроса с системой монет

### Удобство использования
- **Понятные подсказки**: Каждый вопрос имеет кнопку подсказки с указанием стоимости
- **Визуальная обратная связь**: Цветовая индикация правильности ответов (зеленый - правильно, красный - неправильно)
- **Статистика в реальном времени**: Отображение количества правильных ответов и процента правильности
- **Система уровней**: Визуальный прогресс-бар уровня с званиями и иконками
